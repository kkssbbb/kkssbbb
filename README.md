## 🧭 My OOP Philosophy
> “객체지향이란, 객체에게 책임과 역할(행동)을 부여하면서 객체들끼리 협력해서 문제를 해결해 나갈 수 있도록 프로그래밍하는게 객체지향이라고 생각합니다.”

- **행동(Behavior) 우선**: 데이터보다는 행위를 먼저 설계합니다.  
- **역할·책임·협력(R-C-C)**: 한 객체는 한 가지 책임, 협력으로 문제 해결.  
- **Tell, Don’t Ask** 원칙 준수: 객체에게 묻지 말고 일하도록 명령합니다.
